Septembre

Première Semaine

Pour la première semaine, je explore le codebase et me renseigne sur les implémentations existantes du projet

Deuxième Semaine

Au cours de cette semaine, j'ai développé et intégré des évaluateurs simples (Python et JavaScript) de Chainforge dans PromptStudio

  • Modification de la fonction jsonToYml de Chainforge pour gérer les nouveaux cas d'évaluateurs.
  • L'implémentation de l'évaluateur Python
    • Implémentation de execute_python : lancement d'un script d'évaluation via un subprocess avec un wrapper JSON pour récupérer les résultats.
  • L'implémentation de l'évaluateur JavaScript
    • Génération du code JavaScript pour l'évaluateur simple en utilisant createJSEvalCodeFor de Chainforge.
    • Implémentation de l'évaluateur JavaScript en réutilisant la fonction execute_javascript existante.
  • Intégration des évaluateurs dans l'application (fichier worker.ts).

Troisième Semaine

Cette semaine, j'ai ajouté des processeurs de Chainforge dans Promptstudio et j'ai testé l'application sur un cluster.

  • Adaptation de jsonToYml dans Chainforge pour prendre en charge les nouveaux types de processeurs.
  • L'adaptation du base de données pour les processeurs
    • Modification du script SQL : ajout d'une colonne optionnelle format pour les processeurs split et d'une colonne optionnelle code pour les processeurs contenant du code.
    • Création d'un enum dédié pour les types de processeurs (au lieu de réutiliser les types des évaluateurs).
  • Implémentation du processeur Split
    • Développer la fonction execute_split pour le processeur split, en utilisant la fonction splitText de Chainforge.
  • Implémentation du processeur Python
    • Implémenter le processeur Python en réutilisant la fonction execute_python.
  • Raccordement des nouvelles implémentations de processeurs dans worker.ts.
  • Démarrage de l'application sur un cluster
    • Tests sur le cluster (serveur madagh du DIRO) : ouverture d'un tunnel (hole punch) pour exposer MySQL à partir de mon PC à madagh puis validation du fonctionnement — l'application s'est comportée comme attendu sur le cluster.
